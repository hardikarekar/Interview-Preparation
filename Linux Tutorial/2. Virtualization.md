## Use Cases of Virtual Machines (VMs)

* 1. Software Development & Testing
	* Run and test apps on **multiple OS environments** (Windows, Linux, macOS) on one machine.
	- Create **isolated dev environments** without affecting the host system.
	- Snapshot state before risky changes → **easy rollback**.
* 2. Cloud Computing
	- AWS EC2, Azure VMs, Google Compute Engine — cloud providers rent out VMs
	- **Scale up/down** resources on demand
	- Run workloads without owning physical hardware
* 3. Security & Malware Analysis 
	- Open **suspicious files or URLs** inside a VM safely
	- Malware researchers analyze viruses in **sandboxed environments**
	- If the VM gets compromised → just delete it
* 4. Server Consolidation
	- Run **multiple servers** (web, DB, mail) on a single physical machine.
	- Reduces hardware costs and power consumption.
	- Each VM is isolated — one crash doesn't affect others.
* 5. DevOps & CI/CD Pipelines
	- Spin up **clean VMs** for every build/test run
	- Tools like Vagrant automate VM provisioning for consistent environments
	- Basis for containers (Docker runs on a lightweight VM layer)
* 6. Learning & Training
	- Practice **Linux, networking, ethical hacking** without a second machine
	- Platforms like TryHackMe and HackTheBox use VMs heavily
	- Break things freely — just reset the VM
* 7. Legacy Software Support
	* Run **old OS versions** (Windows XP, 32-bit systems) for legacy apps
	- Avoid maintaining old physical hardware
* 8. Network Simulation
	- Simulate **multi-node networks** on one machine (useful for networking labs)
	- Test firewalls, routers, VPNs in isolation
* 9. Disaster Recovery & Backups
	- VM snapshots = instant **system backups**
	- Quickly restore a full system state after failure
	- Easy to **migrate VMs** between physical hosts
* 10. Cross-Platform Gaming / App Compatibility
	- Run Windows games on Linux via a VM
	- Test mobile apps on different OS versions

## How virtualization works
* Virtualization creates an **abstraction layer** between physical hardware and the software running on it — making one physical machine appear as multiple independent machines.
```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│    VM 1     │  │    VM 2     │  │    VM 3     │
│  (Windows)  │  │   (Ubuntu)  │  │  (CentOS)   │
├─────────────┴──┴─────────────┴──┴─────────────┤
│              Hypervisor (VMM)                  │
├────────────────────────────────────────────────┤
│           Physical Hardware                    │
│        (CPU, RAM, Disk, NIC)                   │
└────────────────────────────────────────────────┘
```
### The key component - Hypervisor
* It is software that makes virtualization possible.
* It
	* Sits between hardware and VMs.
	* Allocates resources (CPU, RAM, storage) to each VM.
	* Isolates VMs from each other.
	* Translates VM instructions to physical hardware.
### Two Types of Hypervisors
* Type 1 - Bare Metal (runs directly on hardware)
	* Faster and more efficient
	* Used in enterprise and cloud
	* Examples: VMware ESXi, Microsoft Hyper-V, XEN, KVM
```
┌──────────────────────────┐
│     VM1    │    VM2      │
├──────────────────────────┤
│       Hypervisor         │  ← runs directly on hardware
├──────────────────────────┤
│     Physical Hardware    │
└──────────────────────────┘
```

* Type 2 - Hosted (runs on top of a host OS)
	* Easier to setup - good for desktop use
	* Slight performance overhead
	* Examples: VirtualBox, VMWare workstation, Parallels
```
┌──────────────────────────┐
│     VM1    │    VM2      │
├──────────────────────────┤
│       Hypervisor         │  ← runs as an app
├──────────────────────────┤
│        Host OS           │
├──────────────────────────┤
│     Physical Hardware    │
└──────────────────────────┘
```
